# Contributing

Vielen Dank für dein Interesse am Mitwirken an diesem Projekt.

Kurz: Fork → Branch → Commit → Pull Request.

Details
- Issue anlegen für größere Änderungen oder Diskussionen.
- Branch-Naming: `feature/<kurz-beschreibung>` oder `fix/<kurz-beschreibung>`.
- Tests: Falls du Code änderst, füge wenn möglich einfache Tests hinzu.
- Style: Python 3.7+, type hints bevorzugt, benutze `black`/`flake8` falls vorhanden.

PR-Checkliste
- Eine klare Beschreibung des Problems/Features
- Reproduktionsschritte (falls Bugfix)
- Lokale Tests oder Link zu CI-Run

Kontakt
- Öffentliche Diskussion via Issues oder PR-Review.
